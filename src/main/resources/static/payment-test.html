🧪 결제 테스트

환경 설정

Keycloak 토큰 발급

⚠ userId는 JWT 토큰에서 서버가 추출합니다. 요청 body에 userId를 넣지 않습니다.

Toss 클라이언트 키 설정

⚠ 이 키는 브라우저에서만 사용되며 서버로 전송되지 않습니다.
👤 Customer 권한
STEP 1

결제 생성 + Toss 결제창 POST /api/payments

결제 생성 응답 확인 후 Toss 결제창으로 이동합니다. 완료 후 이 페이지로 돌아와 paymentKey가 자동 입력됩니다.
STEP 2

결제 승인 확인 POST /api/payments/success

Customer 조회

내 결제 단건 조회 GET /api/payments/my/{paymentId}

Customer 조회

내 결제 목록 조회 GET /api/payments/my/reservations/{reservationId}

Customer 조회

내 성공 결제 조회 GET /api/payments/my/reservations/{reservationId}/success

🔑 Admin 권한
Admin 조회

결제 단건 조회 GET /api/payments/{paymentId}

Admin 조회

전체 결제 목록 조회 GET /api/payments

Admin 조회

예약별 결제 목록 조회 GET /api/payments/reservations/{reservationId}

Admin 환불

결제 환불 POST /api/payments/refund/reservations/{reservationId}
